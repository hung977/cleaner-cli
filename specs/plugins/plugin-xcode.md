> **Depends on:** plugins/README, 13 В§12 (worked example B), 14, 16, 19 (staleness), 00 Art. 4/5.

The highest-value developer plugin. Xcode accumulates tensвЂ“hundreds of GB across DerivedData,
device support, module caches, archives, and simulator caches. Risk **varies sharply per
sub-item**: build caches are рџџў, device support рџџЎ, and Archives рџ”ґ (they can be shipped builds).

## 1. Identity

        RootSpec(base: .developer,     glob: "CoreSimulator/Caches/**"),     // рџџў (device runtime handled by simulator plugin)

        RootSpec(base: .developer,     glob: "Xcode/DocumentationCache/**"), // рџџў
        RootSpec(base: .libraryCaches, glob: "org.swift.swiftpm/**"),        // рџџў SwiftPM cache

Scope boundary: touches only `~/Library/Developer/Xcode`, `~/Library/Developer/CoreSimulator/
Caches`, and Xcode's `~/Library/Caches` subtrees. It does **not** touch installed simulator
*runtimes* or *device data* (that is `dev.cleaner.simulator`), never `~/Library/Developer/Xcode/
UserData` (breakpoints, snapshots, themes вЂ” user config), and never a project's own source.

## 2. What it targets

| Sub-item | Path leaf | Why junk / regenerates | Baseline risk |
|---|---|---|---|
| DerivedData (per project) | `DerivedData/<Name>-<hash>/` | Build intermediates, indexes, module cache; regenerated by `xcodebuild`. | рџџў |
| **Stale** DerivedData | as above, project path gone | Orphan вЂ” source project no longer exists. | рџџў (high score) |
| iOS/watchOS/tvOS DeviceSupport | `iOS DeviceSupport/<ver> (<build>)/` | Symbol/debug files per device OS; re-downloaded on next device connect. | рџџЎ |
| Old DeviceSupport | version < N releases behind | Obsolete OS versions you no longer debug. | рџџЎ (higher score) |
| Xcode Archives | `Archives/<date>/<App> <date>.xcarchive` | **May be a shipped/notarized build with dSYMs** вЂ” not regenerable without a rebuild + re-sign. | рџ”ґ |
| IDE caches | `Caches/com.apple.dt.Xcode`, `DocumentationCache`, `swiftpm` | Re-created by Xcode/SwiftPM. | рџџў |

Does **not** target: `UserData/` (user settings), `Products/` inside a live DerivedData of an
open project, provisioning profiles, or the Xcode.app bundle itself (Art. 5 protects `.app`).

## 3. Detection signals & algorithm

**Path scopes the search; metadata + structure justify the action.** Per root:

### 3.1 DerivedData staleness (the key native signal)

Each folder is `DerivedData/<ProjectName>-<28-char-hash>/`. The hash is derived from the project's
absolute path. To confirm safety **without hard-coding**:

1. Read `<folder>/info.plist` if present в†’ `WorkspacePath` key gives the owning project path.
   (Fallback when absent: the folder name's project prefix is a hint only.)
2. `ctx.fs.exists(workspacePath)` вЂ” if the project path **no longer exists** в†’
   `Evidence(kind: .orphaned, detail: "source project gone")`, score raised toward рџџў ceiling.
3. `ctx.metadata.lastUsedDate` / `mtime` of the folder в†’ staleness via spec 19 threshold
   (default 30 days). Recent DerivedData for an *existing* project is still рџџў but *not*
   pre-selected under `--yes` if `mtime < 7 days` (an active build could be mid-flight).
4. `Evidence.isOpenOrLocked` on any `*.xcworkspace`/`build.db` lock inside в†’ skip if Xcode holds
   it (spec 16 В§8): never delete DerivedData for a project Xcode has open.

### 3.2 DeviceSupport obsolescence

Folders named `"<version> (<build>)"`, e.g. `17.5 (21F79)`. Algorithm:

1. Parse the leading semver from the folder name.
2. Determine "current" iOS baseline: read the newest DeviceSupport version present as a proxy,
   and (optional, capability-gated) the shell probe (В§8) for installed SDKs.
3. Folders в‰Ґ `staleMajorVersionsBehind` (default 2) major/minor releases behind в†’ рџџЎ with higher
   score; the newest 1вЂ“2 kept off the default selection (likely your current test devices).
4. `mtime`/`lastUsedDate` corroborates: a DeviceSupport folder untouched > 90 days is stronger
   evidence the device/OS is retired.

### 3.3 Archives (рџ”ґ always)

1. Enumerate `Archives/<date>/*.xcarchive` (directories).
2. Read `Info.plist` inside в†’ `ApplicationProperties` (bundle id, version, build), `Name`,
   `CreationDate`. Presence of a `dSYMs/` and a distribution build в†’ strong "this is a shippable
   artifact" signal.
3. **Never pre-select.** Emit рџ”ґ, `SafetyScore в‰€ 30`, `recoverability = .hard` (rebuild+re-sign),
   rationale naming the app + version. Only surfaced, never auto-cleaned (Art. 4.1).

### 3.4 FindingID

`FindingID = "xcode:<subcategory>:<canonicalPath>"`, e.g. `xcode:derived:/Users/вЂ¦/DerivedData/App-abc`.
Deterministic (DM-7), so incremental scans and rollback correlate across runs.

## 5. Risk & safety scoring

| Sub-item | Risk | Proposed score inputs | Notes |
|---|---|---|---|
| Stale/orphan DerivedData | рџџў 90вЂ“95 | regenerable=yes, user-content=no, orphan=yes, recency old | pre-selected |
| Active DerivedData | рџџў 85вЂ“88 | regenerable=yes, recency recent | shown; deselected under `--yes` if mtime<7d |
| DeviceSupport (old) | рџџЎ 65вЂ“80 | regenerable=yes(cost), device retired | not pre-selected (Art. 4.1) |
| DeviceSupport (current) | рџџЎ 55 | still in use likely | shown, low priority |
| IDE/SwiftPM caches | рџџў 90 | pure cache | pre-selected |
| Archive | рџ”ґ 25вЂ“35 | user-content=maybe(shippable), regenerable=hard | never pre-selected; typed confirm (DM-5) |

The plugin only *lowers* scores from these ceilings with evidence (DM-2); the engine's
`SafetyScorer` (spec 22) has final say. Archives get `recoverability = .hard` в†’ cannot map above
рџ”ґ (DM-1 relation: `.none` forces рџ”ґ; `.hard` keeps it low).

## 6. Recoverability & staging

- All sub-items proposed `Disposition = .stage` (Principle 2) вЂ” even Archives, once confirmed,
  are staged (never purged from a plugin, gate в‘Ј).
- `Recoverability`: DerivedData/caches = `.manual` (rebuild), DeviceSupport = `.manual`
  (reconnect device / re-download), Archives = `.hard` (rebuild + re-sign + re-notarize).
- `RollbackHint`: `restoreFromStaging` with note "DerivedData rebuilds automatically; restore only
  if a build is mid-flight" (spec 13 В§12). Archive hint notes the app/version so a mistaken
  restore is obvious.

## 7. Dry-run / estimate

- `estimate`: streaming `allocatedSize` sum per folder (CC-10), clone/hardlink-corrected. Xcode
  DerivedData rarely shares blocks, so `confidence = .exact`; DeviceSupport may hardlink common
  symbols в†’ `.estimated` when `hardlinkCount > 1` detected.
- `--dry-run` groups by sub-category with per-group totals and the biggest offenders; shows
  Archives in a clearly separated рџ”ґ block that is never in the default selection.

## 8. Shell fallback & its safety

Native metadata is primary; a **read-only** shell probe enriches DeviceSupport/SDK reasoning
where no native API exposes installed SDK versions:

- `context.process.run(["/usr/bin/xcrun", "xcodebuild", "-version"])` and
  `["/usr/bin/xcrun", "--show-sdk-version"]` вЂ” argv array, no shell string (spec 36), 5 s timeout,
  stdout parsed for the current SDK version to anchor "N versions behind." Read-only; failure в†’
  fall back to the newest-present-version heuristic (В§3.2). Never invokes a mutating `xcrun`
  subcommand.
- `xcrun simctl` is **not** used here (owned by `dev.cleaner.simulator`); this plugin only reads
  `CoreSimulator/Caches`, a pure cache path, natively.

Justification (Principle 4 / ADR requirement): there is no first-party Foundation API to list
installed Xcode SDK versions; the probe is the minimal, read-only, timeout-bounded fallback and is
optional (a `.audit`/`.incremental` nicety, not required for a correct scan).

## 9. Edge cases & false-positive mitigations

| Edge case | Mitigation |
|---|---|
| Xcode open with a project building | `build.db`/lock detected `isOpenOrLocked` в†’ skip that DerivedData (spec 16 В§8). |
| DerivedData for a project on an unmounted external volume | `WorkspacePath` unreadable в‰  "gone"; only mark orphan if the *volume* is mounted and the path is absent. Otherwise treat as present-uncertain, рџџў-but-not-preselected. |
| Archive mistaken for junk | Always рџ”ґ, never pre-selected, typed confirm, app/version in rationale. |
| DeviceSupport hardlinks shared across versions | Hardlink cluster accounting (spec 16 В§6) so reclaim isn't overstated. |
| Custom `DerivedData` location (per-workspace or `IDECustomDerivedDataLocation`) | Only the default root is declared; custom locations under `~/Documents`/project dirs are protected and out of scope (documented limitation). |
| `swiftpm` cache shared with command-line SwiftPM builds | Pure cache, re-fetched; safe. |
| Multiple Xcode versions / Xcode-beta | DeviceSupport/DerivedData are shared; version probe uses the active `xcode-select` toolchain. |
| Overlap with `dev.cleaner.devcache` on `~/Library/Caches` | Engine canonical-path de-dup (spec 13 OQ-13.3); this plugin claims the `com.apple.dt.Xcode`/`org.swift.swiftpm` leaves. |

## 10. Test cases

- **T1 orphan DerivedData** (info.plist WorkspacePath в†’ nonexistent) в†’ рџџў, `Evidence.orphaned`,
  pre-selected.
- **T2 active DerivedData, mtime 2 days** в†’ рџџў but deselected under `--yes`.
- **T3 DerivedData with live `build.db` lock** в†’ skipped, `isOpenOrLocked`.
- **T4 DeviceSupport `15.0` with newest present `17.5`** в†’ рџџЎ, "2+ versions behind," not
  pre-selected.
- **T5 `.xcarchive` with distribution build in Info.plist** в†’ рџ”ґ, score в‰€30, `recoverability=.hard`,
  never pre-selected; clean requires typed confirmation.
- **T6 DerivedData on unmounted volume path** в†’ not marked orphan (present-uncertain).
- **T7 xcrun probe times out** в†’ falls back to newest-present heuristic, scan still succeeds.
- **T8 estimate with hardlinked DeviceSupport symbols** в†’ `confidence == .estimated`, shared bytes
  excluded.
