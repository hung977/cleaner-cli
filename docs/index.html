Why cleaner

Native depth, provable safety, and a workflow that fits the terminal.

🛟

Recoverable by default

Cleaned items move to a staging area, not the void. One cleaner undo restores them byte-for-byte.

🎯

You choose

Clean everything, or press s to pick each source with a y/N prompt. Nothing is removed without consent.

⚡

Deep & native

Xcode, SwiftPM, Gradle, npm, Docker, Homebrew, browser & app caches, logs — measured by true on-disk size.

🔒

Protected paths

Documents, Desktop, SSH keys, Keychains and system files can never be deleted — enforced in the engine.

🤖

Scriptable

--yes for automation, --json / --md for machines, a clean exit-code contract for CI.

🔎

More than cleaning

Find large files and byte-identical duplicates read-only: cleaner find large · cleaner find dupes.

Safety first — it's the whole point

A byte wrongly deleted is worse than a gigabyte wrongly kept.

  • You choose — all, pick each, or cancel
  • Moves to staging, not permanent delete
  • cleaner undo restores the last clean
  • Never touches Documents / Desktop / keys
  • Browser cache only — never cookies/passwords
  • --dry-run changes absolutely nothing
  • Append-only audit log of every action
  • Honest sizes — dry-run equals the real run

Commands

One command to clean; a few more when you want them. Full reference →

cleanerScan, then clean all or pick each source (recoverable)
cleaner --dry-runPreview only — clean nothing
cleaner --yesNo prompt — for automation / CI
cleaner undoRestore the last clean
cleaner find largeLargest files (read-only)
cleaner find dupesDuplicate files (read-only)
